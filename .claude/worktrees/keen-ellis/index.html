2025.04 — 2025.11 | 8개월

사내 디지털 리소스 통합 관리 플랫폼

팀 6명 (기획 1 / FE 2 / BE 2 / 서버 1 / 디자인 1) 대상 약 700명 협업 2개 조직
01

문제 정의

사내 디지털 리소스(이미지, 3D 에셋 등)가 조직별로 파편화되어 관리되고 있었으며, 접근 권한 체계가 부재하여 리소스 중복 구매·제작과 보안 리스크가 반복적으로 발생했다.

02

목표

전사 디지털 리소스를 단일 플랫폼에서 통합 관리하고, 메타데이터 표준과 접근 제어 정책을 수립하여 중복 지출을 방지하고 운영 효율을 확보한다.

03

해결 방법

  • 구매/제작 리소스를 단일 창구로 통합하는 프로세스를 구축하여 리소스 탐색 및 온보딩 시간 단축
  • 리소스 출처별 다운로드 승인 워크플로우 및 프로젝트 기반 접근 제어 설계
  • 전사 공통 메타데이터 표준안 수립 및 운영 가이드 제작
04

나의 역할

  • PoC 타겟 인터뷰를 통해 리소스 파편화 및 권한 혼선의 핵심 문제를 도출하고, 상위 구조(메타데이터 표준, 접근 제어 정책, 리소스 플로우)를 기획
  • 운영 부서 간 요구사항 조율을 기반으로 전사 공통 가이드 및 온보딩 문서를 제작하여 신규 플랫폼 초기 안착에 기여
  • 표준안 수립 과정에서 조직 간 우선순위가 엇갈릴 때, 리소스 접근성과 보안 수준을 의사결정 기준으로 제시해 합의를 이끌어냄
05

결과

  • 리소스 중복 제작 및 구매율 감축
  • 수동 운영 공수 최소화 및 보안 리스크 차단
  • 신규 플랫폼 전사 안착 달성
06

인사이트

다수 조직이 관여하는 플랫폼은 기술 설계보다 '운영 합의'가 선행되어야 한다. 표준안 수립 과정에서 이해관계자의 공감을 먼저 확보한 것이 전사 안착의 핵심이었다.

2024.09 — 2025.03 | 7개월

성우 녹음 플랫폼

팀 3명 (기획 1 / FS 1 / 디자인 1) 대상 약 500명 협업 4개 조직
01

문제 정의

조직마다 녹음 요청 프로세스와 데이터 입력 방식이 상이하여 녹음 공정의 표준화가 불가능했으며, 반복적인 재녹음으로 인해 제작 비용과 리드타임이 지속적으로 증가하고 있었다.

02

목표

국내외 녹음 공정을 표준화하고, AI 기술을 결합한 자동화 시나리오를 설계하여 제작 비용과 리드타임을 대폭 절감한다.

03

해결 방법

  • 조직별로 상이하던 녹음 요청 데이터 입력 표준안을 수립하여 전사 공통 녹음 관리 워크플로우 설계
  • AI(TTS 및 Facial Animation) 기술을 녹음 공정에 결합한 자동화 시나리오 기획으로 반복적인 재녹음 공정 최소화
  • 주요 성공 사례(Best Case) 정의 및 전사 확산 전략 수립
04

나의 역할

  • 전사 공통 녹음 관리 워크플로우 설계 및 데이터 입력 표준안 수립
  • AI 조직과 프로젝트 팀 간 복잡한 요구사항을 기술적 실현 가능성 중심으로 조율하고, 체계적인 의사결정 구조를 수립하여 플랫폼 구체화
  • 요구사항이 상충할 때 실현 가능성과 비용 대비 효과를 기준으로 우선순위를 정해 이해관계자 합의를 이끌어냄
  • Best Case 정의 및 전사 확산 전략을 수립하여 녹음 필요 조직 대부분에 플랫폼 안착
05

결과

  • 제작 비용 30% 이상 절감
  • 제작 리드타임 40% 단축
  • 녹음 필요 조직 대부분에 플랫폼 안착, 운영 리소스 최소화
06

인사이트

AI 기술 도입은 기술 자체보다 '기존 공정과의 결합 시나리오'를 얼마나 구체적으로 설계하느냐에 성패가 달린다. 현장의 실제 워크플로우를 깊이 이해한 후 자동화 포인트를 선정한 것이 높은 비용 절감으로 이어졌다.

2024.03 — 2024.07 | 5개월

전사 통합 용어 관리 플랫폼 (TMS)

팀 4명 (기획 1 / FE 1 / BE 1 / 디자인 1) 대상 전사 4,000명 협업 2개 조직
01

문제 정의

용어 관리가 엑셀 기반 수기 업로드 방식으로 운영되어 데이터 누락과 휴먼 에러가 빈번했으며, 번역된 용어의 재사용 체계가 없어 중복 번역 요청이 반복되고 서비스 용어 일관성이 확보되지 못했다.

02

목표

분산된 용어 관리 체계를 전사 통합하고, 번역 플랫폼(MemoQ)과의 연계를 통해 중복 번역을 최소화하며 서비스 용어 일관성을 확보한다.

03

해결 방법

  • 엑셀 기반 수기 업로드 방식을 MemoQ API 연동 체계로 전환하여 데이터 누락 및 휴먼 에러를 근본적으로 차단
  • '플랫폼 용어 등록 → MemoQ 번역 수행 → 확정 데이터 자동 동기화' 프로세스 설계로 번역 데이터의 최신성 및 신뢰도 확보
  • 이미 번역된 용어의 자동 호출 및 재사용 구조를 구축하여 불필요한 중복 번역 요청 최소화
04

나의 역할

  • 개발 조직의 기술 문서를 기반으로 용어 등록 / 번역 / 동기화 프로세스를 설계하고 수기 운영 방식을 API 연동 체계로 전환
  • 현지화 조직의 요구사항을 정리하고 화면 설계 및 워크플로우 설계를 통한 서비스 구체화
  • 실무 매뉴얼 배포 및 운영 교육을 통해 신규 플랫폼으로 전환 지원
05

결과

  • 중복 번역 요청 30% 감소
  • 서비스 용어 일관성 확보
  • 전체 제작 리드타임 단축
06

인사이트

레거시 시스템 전환의 핵심은 현업 사용자의 저항을 최소화하는 것이다. 개발 조직의 기술 문서를 기반으로 운영 프로세스를 설계하고, 현지화 조직과의 공통 기준을 수립한 것이 전환 성공의 핵심이었다.

핵심역량

문제 정의

PoC 인터뷰와 이해관계자 분석을 통해 표면적 요청 이면의 핵심 문제를 도출하고, 해결 가능한 범위로 구조화합니다.

요구사항 구체화

기술 문서와 비즈니스 요구를 연결하여 실현 가능한 프로세스로 구체화하고, 조직 간 커뮤니케이션 비용을 최소화합니다.

프로세스 설계 및 표준화

전사 공통 워크플로우를 설계하고 운영 체계를 구축하여 조직 간 일관된 프로세스를 정착시킵니다.

Cross-Functional 협업

개발·디자인·AI·현지화 조직 간 복잡한 요구사항을 기술적 실현 가능성 중심으로 조율하고 의사결정 구조를 수립합니다.

내부 플랫폼 기획

사내 서비스 기획부터 전사 확산 전략까지 End-to-End로 설계하여 플랫폼의 초기 안착과 지속적 성장을 이끕니다.
